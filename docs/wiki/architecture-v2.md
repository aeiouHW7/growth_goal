## 三、人生档案（M2）详细设计

### 3.0 四层结构（已确认）

```
LifeArchive（每个用户一份，1:1 关联 User）
│
├── 层1：价值观 Values
│   写入方式：AI 引导建立 / Web 手动编辑
│   内容：vision, coreValues, priorities, boundaries
│         + MBTI + 大五OCEAN(5分数) + 盖洛普Top5
│   特点：几乎不变
│
├── 层2：能力与资源现状 Capabilities
│   写入方式：Web 表单（固定问题，不需要AI）
│   内容：职业/收入、时间资源、财务、技能自评、健康、限制条件
│   特点：客观事实，用户自己填
│
├── 层3：历史行为数据 BehaviorHistory
│   写入方式：AI 从每日复盘中提取
│   内容：行为模式总结、自我认知列表、偏误趋势、能力变化趋势
│   特点：持续积累，AI 维护
│
├── 层4：未来蓝图与目标 Blueprint
│   写入方式：Web + AI 辅助
│   内容：复用现有 LifeGoal/YearlyGoal 体系，档案中只存摘要引用
│   特点：动态变化
│
└── AI 摘要（跨层）
    │   summary — AI 自动生成的统一摘要（<200 字）
    │   任何一层有变化时重新生成
    │   每次 Claude CLI 对话前加载
```

### 3.1 层1：价值观 — 数据结构（已确认）

```prisma
model LifeArchive {
  id            String   @id @default(uuid())
  userId        String
  user          User     @relation(fields: [userId], references: [id])

  // ── 层1：价值观 ──
  vision        String?  // 理想生活描述（自由文本）
  coreValues    String?  // 核心价值观（自由文本）
  priorities    String?  // 优先级说明
  boundaries    String?  // 底线/不可牺牲

  // ── 层1：人格测评 ──
  mbtiType      String?  // MBTI 4位代码，如 "INFJ"

  bigFive_O     Float?   // 大五开放性 1-10
  bigFive_C     Float?   // 大五尽责性 1-10
  bigFive_E     Float?   // 大五外向性 1-10
  bigFive_A     Float?   // 大五宜人性 1-10
  bigFive_N     Float?   // 大五神经质 1-10

  gallupTop5    String[] // 盖洛普优势 Top5，如 ["战略","学习","理念"]

  // ── 层2：能力与资源（待确认字段）──
  // ── 层3：历史行为数据（待设计）──
  // ── 层4：未来蓝图摘要（待设计）──

  // ── AI 摘要 ──
  summary       String?  // AI 自动生成的统一摘要，供对话加载

  createdAt     DateTime @default(now())
  updatedAt     DateTime @updatedAt
}
```

### 3.2 人格分析引擎（待设计）

```
数据库存的是原始数据：          后端翻译后给 Claude 的是：

MBTI: INFJ        ──────────→  【MBTI: INFJ — 理想主义者】
BigFive: C=8,N=3  ──────────→  · 主导功能Ni：从愿景出发做决策
Gallup: 战略学习  ──────────→  · 高自律(C=8)，不需要外部推力
                                · 情绪稳定(N=3)，可直接反馈
                                · 喜欢框架式学习(战略+学习)
                                · 盲区：容易想太多不落地
```

**这个分析引擎需要翻译 3 套框架：**

| 框架 | 输入 | 输出（给 Claude 的洞察文本） |
|------|------|---------------------------|
| MBTI 翻译器 | "INFJ" | 认知功能堆叠、沟通风格、决策偏好、盲区 |
| OCEAN 翻译器 | C=8,N=3... | 能力映射、干预策略建议、反馈方式偏好 |
| 盖洛普翻译器 | ["战略","学习"] | 优势应用建议、动机来源、目标设计偏好 |

设计要点：
- 纯后端逻辑，无需调 Claude CLI
- 输入是原始分数/代码，输出是结构化文本
- 输出直接嵌入到 Claude CLI 的 Prompt 上下文中
- 规则基于心理学框架的固定映射（不是 AI 自己猜）

### 3.3 数据流（已确认）

```
用户输入（CLI/IM/Web）
  → 后端 API
    → 读取 LifeArchive（4层）+ 目标 + 复盘
    → 调用人格分析引擎（原始分数→行为洞察）
    → 组装统一 Prompt 上下文（<500 字）
    → 发给 Claude CLI
    → Claude 基于完整上下文做分析/建议
    → 返回结果
    → 后端 ArchiveGate 判断是否需要更新档案
    → 如有更新 → 重新生成 summary
```

### 3.4 建立流程（新用户首次）

**方式A：AI 引导对话（主要方式，Claude CLI）**
- Claude CLI 用固定引导问题收集信息
- 输出结构化字段 → 写入 LifeArchive

**方式B：手动填写（Web 页面，跳过 AI）**
- 用户直接填表单
- 直接写入 LifeArchive

### 3.5 更新机制

```
触发更新的事件（后端 ArchiveGate 判断）:
├── L1 信号 — 直接写入（无需 AI）
│   · 用户在 Web 手动修改
│   · 用户明确说"记下来"
│
├── L2 信号 — 调 Claude CLI 提取结构化内容
│   · 用户在复盘中表达新的自我认知（"我发现我更适合..."）
│   · 用户做了重大决策（"我决定不做了"）
│   · 用户生活状态变化
│
├── L3 信号 — 定期自动蒸馏
│   · 每周/每月，聚合近期复盘数据，提炼新洞察
│   · 由后端 Pattern 检测触发
│
└── 不触发（仅当前对话使用，不持久化）
    · 普通日常聊天内容
    · 情绪发泄
    · 重复已有认知的内容
```

### 3.6 读取机制（每次 AI 对话前）

```
后端组装 buildUserContext(userId) → 输出文本块（< 500 字）：

=== 用户画像上下文 ===
【基础】AI产品经理，工作日可用3.7h，美容院装修中
【价值观】时间自由 > 家庭幸福 > 健康 > 创造
【人格】INFJ（从愿景出发）
       大五 C=8(高自律) N=3(情绪稳定)
       盖洛普：战略、学习、理念
【当前目标】体脂率20%（进度22%→20%）
           自媒体500粉（59/500）
【近期模式】视频收尾卡顿×3（第4天）

→ 每次 Claude CLI 调用时自动注入
→ Claude 不需要自己查数据库，直接基于此做分析
```
