# 目标拆解引擎

> 被 `agents/coach.md` 引用。当输入类型为"愿望/想法"时加载。

---

## 核心原则

- **愿景驱动**：不是管理用户给的 todo，而是理解他想成为什么样的人，倒推需要做什么
- **校准历史**：设定目标时必须参考用户的行为模式和能力基线，避免"做了也白做"
- **小步启动**：宁可拆到「不可能不做」的程度，也不要一步跨太大

---

## B1: 理解人生愿景（Design Your Life 方法论）

### 引导问题（按顺序，一次只问一个）

1. 「你理想的一天是什么样的？从起床到睡觉，描述一下。」
2. 「如果不需要考虑现实限制，你最想成为什么样的人？」
3. 「十年后你希望自己在哪里、在做什么、和谁在一起？」
4. 「什么对你来说最重要——自由？成就？关系？安稳？」

### 输出

用户回答后，总结为一段清晰的人生愿景描述。示例：

> 用户说："我想过能自由选择上班、自由选择休假、家庭幸福的生活"
> → 总结：「你的人生愿景是**时间自由、财务安全、家庭幸福**——你有足够的被动收入覆盖基本生活开支，可以自主安排工作节奏，同时有质量地陪伴家人。」

**写回**：通过 `PUT /api/user` 更新 `goalDomains` 和 `lifeVision` 字段。

---

## B2: 从愿景拆解人生领域

### 从愿景反推框架

| 领域 | 从愿景反推的问题 |
|------|----------------|
| 💼 事业/职业 | 什么样的工作方式能实现你的理想状态？当前主业是跳板还是枷锁？ |
| 💰 财务 | 需要多少被动收入才能自由？当前资产负债结构如何？ |
| 🏋️ 健康 | 什么样的身体状态能支撑你追求愿景？当前习惯是否匹配？ |
| 👨‍👩‍👧 家庭 | 你理想的家庭生活是什么样的？当前差距在哪？ |
| 📚 成长 | 需要什么核心能力才能实现这个愿景？ |
| 🚀 副业/创作 | 哪些方向能加速实现你的目标？ |
| 🤝 社交 | 什么样的人脉网络能帮助你？ |

**规则**：不是每个领域都要填。只选与用户愿景直接相关的领域（通常 3-5 个）。

### 输出

用户确认后，通过 `PUT /api/user` 更新 `goalDomains`。

## B3: 设定长期目标（3-10年）

### OKR 框架：每个领域 1 个 O + 3 个 KR

- **O（目标）**：定性、有野心、方向性
- **KR（关键结果）**：定量、可衡量、可验证

示例：
```
领域：财务
O：5年内被动收入覆盖月基本生活开支
KR1：2028年副业月收入达到8000元
KR2：建立2个稳定变现渠道
KR3：核心投资资产达到30万

领域：副业/自媒体
O：3年内建立起可持续的内容副业
KR1：小红书粉丝达到5万
KR2：内容收入+引流收入达到5000元/月
KR3：形成可持续的内容生产流程（每周≤10小时）
```

### 校准规则

设定目标前，必须查询：
- `GET /api/analysis/patterns/recurring`：如果有「知道但不做」模式频率≥3 → 建议更小步的里程碑
- `GET /api/analysis/capabilities`：能力短板领域的目标应配套能力培养计划

## B4: 拆解年度目标（Backward Goal Setting + WOOP）

### 逆向倒推

```
5年后目标                          ← B3 定的长期目标
    ↓ 倒推
2026年底要达到什么程度              ← 第一个年度目标
    ↓ 倒推
今年能做什么，进度怎么安排
```

### WOOP 障碍识别

设定每个年度目标时，过一遍：

| 步骤 | 问题 | 用途 |
|------|------|------|
| **W**ish | 今年在这个领域你最想达到什么？ | 明确目标 |
| **O**utcome | 如果达成，最好的结果是什么？ | 激发动机 |
| **O**bstacle | 什么会阻碍你？你的历史模式中有什么风险？ | 识别障碍 |
| **P**lan | 如果[障碍]发生，你就[行动] | 制定预案 |

### SMART 验证

每个年度目标必须过 SMART：
- **S**pecific（具体）— 不是"搞副业"而是"小红书粉丝10000"
- **M**easurable（可衡量）— 有数字指标
- **A**chievable（可达成的）— 结合用户时间资源
- **R**elevant（相关的）— 必须和长期目标/愿景直接关联
- **T**ime-bound（有时限的）— 有明确截止时间

### 字段

```json
{
  "metricType": "NUMERIC | DURATION | FREQUENCY | PERCENTAGE | STAGE",
  "targetValue": "目标值",
  "startValue": "起始值"
}
```

## B5: 拆解月度计划

每个年度目标拆成 12 个月度计划。

### 规则

- **第一个月的计划尤其具体**：降低启动门槛是执行成功的关键
- **每个计划要可验证**：有明确的完成标准
- **考虑用户时间可用性**：工作日可用约3.7h/天，周末约6h/天

### 示例

```
年度目标：小红书粉丝达到10000
  └── 第1月：内容体系搭建
        - metricType: STAGE
        - targetValue: "定位确认 + 10篇内容储备"
  └── 第2月：稳定输出
        - metricType: FREQUENCY
        - targetValue: "12篇/月（周更3篇）"
  └── 第3月：增长优化
        - metricType: NUMERIC
        - targetValue: "1000粉丝"
```

## B6: 每日任务（Tiny Habits + Implementation Intentions）

### 最小行动原则

把任务拆到「不可能不做」的程度：

```
❌ "每天写一篇高质量文章"   → 太大，执行门槛高
✅ "每天早上倒水后写3句话" → 小到不可能拒绝
```

### If-Then 计划

```
格式：如果[情境]，那么[行动]

示例：
"如果早上喝完咖啡，就打开编辑器写5分钟"
"如果晚饭后坐下，就打开本子列明天的3件事"
```

### 任务粒度

- 单个任务 ≤ 30 分钟可完成
- 每个任务有明确的完成信号（可以打勾）
- 优先保下限（每天最低完成量），不追求上限

## B7: 写入 API

创建目标的顺序：

```bash
# Step 1: 如果确认了新的人生愿景，更新用户
curl -s -X PUT http://localhost:3001/api/user \
  -H 'Content-Type: application/json' \
  -d '{"goalDomains": ["财务","副业","健康","家庭","成长"]}'

# Step 2: 创建人生目标
curl -s -X POST http://localhost:3001/api/goals/life \
  -H 'Content-Type: application/json' \
  -d '{"title":"时间自由——被动收入覆盖基本生活开支","description":"5年内...","timeHorizon":"5年"}'

# Step 3: 创建年度目标（关联上面的人生目标ID）
curl -s -X POST http://localhost:3001/api/goals/yearly \
  -H 'Content-Type: application/json' \
  -d '{"userId":"<ID>","lifeGoalId":"<ID>","title":"小红书粉丝10000","year":2026,"metricType":"NUMERIC","targetValue":"10000","startValue":"0"}'

# Step 4: 创建月度计划（逐月）
curl -s -X POST http://localhost:3001/api/plans/monthly \
  -H 'Content-Type: application/json' \
  -d '{"userId":"<ID>","yearlyGoalId":"<ID>","title":"内容体系搭建","year":2026,"month":7,"metricType":"STAGE","targetValue":"定位确认+10篇内容储备"}'

# Step 5: 创建日计划（可选）
curl -s -X POST http://localhost:3001/api/plans/daily \
  -H 'Content-Type: application/json' \
  -d '{"userId":"<ID>","title":"写一篇小红书笔记","date":"2026-06-07","metricType":"NUMERIC","targetValue":"1"}'
```

**注意**：创建人生目标时，会返回一个 `id`，后续的年度目标需要用这个 `id` 作为 `lifeGoalId`。类似地，年度目标的 `id` 用于月度计划的 `yearlyGoalId`。

---

## B8: 拆解验证

创建后，调用 `GET /api/progress/overview` 确认目标已创建成功。

向用户展示完整的目标链（愿景 → 长期目标 → 年度目标 → 月度计划），让用户确认。
