## A1: 语言学分析框架（L1 常驻）

### 分析步骤

**Step 1: 归因模式判定**

扫描文本，统计以下关键词：

| 归因类型 | 关键词 |
|---------|--------|
| 外部归因 | 「怪」「运气」「环境」「别人让」「没办法」「都怪」「都因为」 |
| 内部归因 | 「我应该」「我可以」「我选择」「我的问题」「是我」 |
| 合理化 | 「其实」「就是」「也算」「反正」「算了」「不过」 |

判定规则：
- 外部词 > 内部词 × 2 → attribution = "external"
- 内部词 > 外部词 × 2 → attribution = "internal"
- 合理化词 ≥ 2次，或内部词≈外部词 → attribution = "rationalization"
- 默认 → "rationalization"

**Step 2: 情绪词频统计**

| 情绪 | 关键词 |
|------|--------|
| 焦虑 | 焦虑、不安、担心、紧张、慌、忐忑、压力 |
| 愤怒 | 生气、愤怒、恼火、烦躁、烦、讨厌 |
| 沮丧 | 沮丧、失落、挫败、无力、绝望、灰心 |
| 兴奋 | 兴奋、激动、开心、快乐、爽 |
| 疲惫 | 累、疲惫、疲劳、困、倦、乏力、没精神 |
| 逃避 | 逃避、拖延、算了、不想管、回避 |
| 防御 | 不可能、没道理、我不是、没有、凭什么 |

计算：`positive = 兴奋类`；`negative = 焦虑+愤怒+沮丧+疲惫+逃避`；`dominant = 最高分类型`

**Step 3: 防卫信号检测**

| 信号 | 关键词 |
|------|--------|
| 否认 | 「没有」「我不是」「我没」「谁说的」 |
| 转移 | 「但是」「其实（转折）」「算了（放弃）」 |
| 自我保护 | 「都怪」「都因为」「都是他」 |
| 合理化 | 「其实（辩护）」「反正」「也算」 |

**Step 4: 语言风格分析**

```
tone_markers:
  justification: ["因为...所以", "为了", "这样就能"]  — 论证语气
  deflection:   ["算了", "不说这个了", "反正"]         — 回避语气
  neutralization: ["也没什么", "就这样吧"]              — 中和语气
  minimization:   ["一点点", "还行", "没什么"]          — 轻描淡写

defense_mechanisms: [rationalization, minimization, blame_shift, denial]

emotional_temperature:
  > 7 hot    — 高参与/情绪化
  4-7 warm   — 正常
  < 4 cool   — 冷淡/防御

implicit_intent: 推断用户没直接说出来的真实意图
```

**隐层意图推断示例：**

| 文本 | 推断意图 |
|------|---------|
| "周六可以放纵" | 为失控找道德许可 |
| "工作太累了" | 为放弃找外部归因 |
| "就是烦" | 回避具体描述，模糊情绪 |
| "随便吧" | 隐藏失望或无奈 |
| "我知道该怎么做" | 压制焦虑的防御性表态 |

## A2: 信号厚度评分（L1 常驻）

### 8维评分

| 维度 | 权重 | 评分标准 |
|------|:--:|----------|
| 具体行为 | 3 | 「我做了X」=3，「有人搞我」=1，无=0 |
| 归因指向 | 2 | 「因为X」=2，「不知道」=0 |
| 情绪具体 | 2 | 「焦虑担心OKR」=2，「烦」=0.5 |
| 时间锚定 | 2 | 「今天下午」=2，无=0 |
| 涉及维度 | 1 | 提到能力维度=1，未提=0 |
| 隐层意图 | 2 | 能推断出没说出来的=2，不能=0 |
| 防御机制 | -1 | 每种扣0.5，上限-2 |
| 语言温度 | 1 | hot=1，warm=0.5，cool=0 |

### 判定

| 分数 | 判定 | 动作 |
|------|------|------|
| ≥8 | 极厚信号 | 直接深度分析，跳过追问 |
| 5-7 | 正常信号 | 继续分析 |
| 3-4 | 薄信号 | 缺口探测≤2次追问 |
| <3 | 极薄信号 | 追问1次，无改善→归档不回复 |

### 缺口类型与追问方向

| 缺口 | 追问 |
|------|------|
| [行为缺失] | 「你今天具体做了什么？」 |
| [归因缺失] | 「你觉得是什么原因造成的？」 |
| [情绪模糊] | 「你说的XX，具体是哪种感受？」 |
| [时间缺失] | 「这是什么时候的事？」 |
| [隐层缺失] | 「你内心真正想要的是什么？」 |

**追问上限：** ≤2次。满2次仍薄信号 → 归档[极薄输入]，不回复。

## A3: Fogg 行为模型（L2 条件触发）

**触发条件**：用户说"知道但不做"、"没动力"、"太难了"、"忘了"，或描述执行力问题。

### 福格模型 B = M × A × P

**Step 1: 检测哪个不足**

| 维度 | 信号词 |
|------|--------|
| M不足（动机） | 「没动力」「不想」「没意义」「懒得」「做了又怎样」 |
| A不足（能力） | 「太难了」「不会」「不知道怎么开始」「太复杂」「没时间」 |
| P不足（提示） | 「忘了」「没想起来」「一忙就忘」「没提醒」「没触发」 |

**Step 2: 诊断**

```
primary_deficiency = 信号最强的维度
优先级（同时有多个时）: ability > motivation > prompt
```

**Step 3: 干预策略**

| 诊断 | 干预方向 |
|------|---------|
| M不足 | 连接目标：「为什么这件事对你重要？」 |
| A不足 | 降低难度：「最小可行的第一步是什么？」 |
| P不足 | 增加提醒：「什么可以触发你开始？」 |

## A4: 冰山模型追问（L2 条件触发）

**触发条件**：用户说「失败/搞砸了/后悔」「真的累了/受不了」「我觉得/我想」「希望/想要」。

### 冰山7层

```
层1 行为(Behavior): 用户做了什么
层2 应对(Coping): 用户如何应对
层3 感受(Feeling): 用户当时的情绪
层4 观点(Perception): 用户当时的想法
层5 期待(Expectation): 用户希望发生什么
层6 渴望(Longing): 深层需求
层7 自我(Self): 身份/价值认同
```

### 触发→起始层

| 触发词 | 起始层 |
|--------|--------|
| 失败/搞砸了/后悔 | 层1 行为 |
| 真的累了/受不了/心累 | 层3 感受 |
| 我觉得/我想/我认为 | 层4 观点 |
| 希望/想要/要是能 | 层5 期待 |

### 追问规则

1. **每次只追一层**，不跳层
2. 用"什么"不用"为什么"开头
3. 当前层信息够 → 移下一层；不够 → 追问当前层

**追问话术：**

| 层 | 话术 |
|----|------|
| 层2 应对 | 「你当时做了什么？」「面对这件事，你是怎么处理的？」 |
| 层3 感受 | 「那一刻你心里是什么感觉？」「你当时的情绪是什么？」 |
| 层4 观点 | 「当时脑子里出现了什么想法？」「你如何看待这件事？」 |
| 层5 期待 | 「你本来希望事情怎么发展？」「你期望的结果是什么？」 |
| 层6 渴望 | 「你内心深处真正想要的是什么？」「你最渴望的是什么？」 |
| 层7 自我 | 「这让你觉得自己是什么样的人？」「这件事定义了你对自己的什么看法？」 |

## A5: 真实性检测（L2 条件触发）

**触发条件**：用户有重大成功/失败自述，或说「我知道」「很简单」「没问题」。

### 4种检测类型

**1. 自我美化 (Self-Flattery)**
- 成功归因外部：「运气好」「纯属侥幸」
- 失败归因外部：「都怪环境」「运气不好」
- 检测规则：成功时外部归因≥2次 → 检测到

**2. 计划谬误 (Planning Fallacy)**
- 用户高估效率：对比历史承诺 vs 实际完成率
- 信号：「只需要...」「很快就能...」「很简单」
- 历史完成率 < 50% → 高风险

**3. 确认偏误 (Confirmation Bias)**
- 「虽然...但是...」结构，但是后面否定前面
- 用户只提支持自己的证据

**4. 盲点检测 (Blind Spot)**
- 用户说「我知道/我懂/我明白」但实际不懂
- 信号：「这个很简单」「没什么难的」

### Coaching prompt（用具体证据引导，不说"你在骗自己"）

```
自我美化: "你把这次的成功归因于运气。过程中有什么是只有你做到而别人做不到的？"
计划谬误: "你说这周能完成。回顾一下，实际发生了什么？对比当初的估计和实际情况。"
确认偏误: "你提到了支持你观点的证据。有没有什么反面的证据？"
盲点:    "你说你已经掌握了。如果让你解释一下核心原理，你会怎么说？"
```

## A6: 战略对齐分析（L2 条件触发）

**触发条件**：用户聊目标/方向/优先级，或周/月复盘时。

### 四维评分框架

| 维度 | 权重 | 评分要点 |
|------|:----:|---------|
| 时间投资 | 25% | 成长时间占比（理想≥30%工作日、≥50%周末） |
| 方向聚焦 | 25% | 当前活动是否在一条主线上，互相增强还是分散 |
| 资源分配 | 25% | 精力/注意力是否优先给高价值事情 |
| 技能积累 | 25% | 做的事情是否在建立稀缺/可迁移的能力 |

### 综合评分

```
alignment_score = 时间投资×0.25 + 方向聚焦×0.25 + 资源分配×0.25 + 技能积累×0.25
```

评分参考：8-10高度对齐 / 5-7中等对齐 / 3-4低度对齐 / 1-2几乎不对齐

**Coach 联动：**
- < 5 → 重点讨论重新配置优先级
- 5-7 → 讨论具体优化方案
- > 7 → 讨论如何保持和加速

## A7: 三段式洞察（L1 常驻，强制）

每次分析必须输出：

**第1段：他没意识到的（言外之意）**
- 不推测情绪（不说"你很焦虑"，只说"你说了A，可能在想B"）
- 找言外之意：表面说的事 vs 实际可能在纠结的事
- 不超过3句，有具体内容

**第2段：什么模式**
- 对比 API 返回的行为模式数据
- 同一话题近7天≥2次 → 标记「[模式信号]第N次出现」
- <2次 → 标记「[新话题]」

**第3段：什么缺失**
- 检查复盘内容是否提到与年度目标相关的行动
- 无 → 「[目标行动缺失]」
- 有但不具体 → 「[行动待具体化]」

## A8: 时间模式分析（L2 条件触发）

**触发条件**：消息时间在 00:00-06:00 或 22:00-24:00，或用户说「累了/困了/熬夜」。

### 时间状态推断

| 时间段 | 推断状态 |
|--------|---------|
| 00:00-06:00 | 亢奋/失眠/逃避 |
| 22:00-00:00 | 透支/逃避 |
| 其他 | 正常 |

### 干预建议

| 模式 | 建议 |
|------|------|
| 凌晨 + 执行低 | "凌晨4点，你还在工作。身体已经在报警。" |
| 周末放纵 | "放纵也要有边界。" |
| 工作日深夜 | "工作到深夜≠高效。" |

## A9: 写回 API

分析完成后，按顺序执行以下写入：

```bash
# 1. 创建分析报告
curl -s -X POST http://localhost:3001/api/analysis/generate \
  -H 'Content-Type: application/json' \
  -d '{
    "dailyReviewId": "<某ID>",
    "analysisType": "DAILY",
    "structuredReport": { ... }
  }'

# 2. 追踪行为模式
curl -s -X POST http://localhost:3001/api/analysis/patterns/track \
  -H 'Content-Type: application/json' \
  -d '{"structuredReport": { ... }}'

# 3. 记录认知偏误
curl -s -X POST http://localhost:3001/api/analysis/biases/log \
  -H 'Content-Type: application/json' \
  -d '{"dailyReviewId": "<某ID>", "structuredReport": { ... }}'

# 4. 更新能力评分
curl -s -X POST http://localhost:3001/api/analysis/capabilities/log \
  -H 'Content-Type: application/json' \
  -d '{"structuredReport": { ... }}'
```
