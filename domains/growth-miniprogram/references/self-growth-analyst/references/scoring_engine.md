# Scoring Engine (评分引擎)

评分不是自动的。评分是证据的结晶。

---

## 基本原则

**没有证据的评分是幻觉。**

每次评分必须：
1. 有具体行为记录（不是感觉）
2. 有对比（和上次比是进步、退步还是原地）
3. 有至少2条引用来源（日期+内容摘要）

---

## 评分时间表

| 类型 | 触发时间 | 更新内容 |
|------|---------|---------|
| 周考微调 | 每周日晚 | 只调整本周有明确变化的维度，其他不动 |
| 月度评分 | 每月最后一天 | 完整20维评分 + 分析 + 下月重点 |
| 半年总结 | 每6个月 | 趋势分析 + 模式变化 + 重大复盘 |

---

## 评分标准（每维度1-10分）

每个维度有同样的标准：

| 分数 | 含义 | 行为描述 |
|------|------|---------|
| 1-3 | 严重短板 | 这个能力正在拖累你，必须立即正视 |
| 4-5 | 明显缺失 | 知道概念，但行为上没有持续体现 |
| 6 | 及格线 | 有意识在做，但不稳定，时好时坏 |
| 7-8 | 优秀 | 稳定输出，能主动调用这个能力 |
| 9-10 | 顶尖 | 这个能力已经成为你的标签，别人会向你学习 |

**预警线：6分以下必须标注为"危险区"**

---

## 证据引用格式

每次评分必须包含这个格式：

```
### [维度名称]：X → Y (变化方向)

**证据1（来源：YYYY-MM-DD）**
> [具体引用内容]
> 分析：这句话/行为为什么说明了这个能力的变化

**证据2（来源：YYYY-MM-DD）**
> [具体引用内容]
> 分析：同上

**AI观察**
> 我看到了什么变化（或没有变化）

**用户确认**
> [月考对话时，用户自己怎么说]

**评分结论**
> 为什么给这个分，而不是更低或更高
```

---

## 周考微调规则

**调整条件（满足任一）：**

1. 本周有重大突破——某个维度有明显的新的主动行为
2. 本周有明显失误——某个维度出现了严重的倒退行为
3. 连续两周都没变化的维度——可以降分（因为没有进步就是退步）

**不调整条件：**
- 波动很正常，不因为一周的表现改变大方向
- 除非有明确的突破或失误，否则维持

**调整幅度：**
- 单次最多 ±1 分
- 要有证据才能动，不能凭感觉

## 月度评分流程

**Step 1：汇总本月所有记录**

加载：
- 本月所有碎碎念
- 本月所有周考记录
- 本月任何主动告诉我的信息

**Step 2：按维度分析**

对每个维度，逐一检查：
1. 这个月出现了几次相关碎碎念？
2. 行为有没有变化？
3. 和上月比是进步还是退步？
4. 证据是什么？

**Step 3：打分（每个维度）**

按证据格式写，然后用 1-10 分打分。

**Step 4：对比上月**

输出格式：
```
## [维度] 评分变化

上月：X分 → 本月：Y分
变化：↑/↓/—
证据：...
结论：...
```

**Step 5：识别最需要突破的维度**

月度评分后，必须明确指出：
1. 最低的3个维度（危险区）
2. 最容易突破的1个维度（投入产出比最高）
3. 用户最应该投入时间的维度

---

## 评分与用户对话

月度评分后，必须和用户有一段对话。

**对话目的：**
1. 确认AI的判断是否准确（用户可能不同意）
2. 追问AI有疑问的地方
3. 让用户自己说说他认为的变化

**对话规则：**
1. 先说结论，再问"你觉得呢"
2. 不强迫用户接受评分，但要有证据支撑
3. 如果用户反对，问"你的理由是什么"，然后记录

**月度对话的模版：**

```
本月的评分出来了。

[维度]：X分（上月Y分）
证据：...（2条）

[维度]：X分（...）
...

我最想问你的是：
1. 你同不同意[最低维度]的打分？你觉得问题在哪？
2. [最需要突破的维度]，你准备怎么做？

你可以反驳我，但要有理由。
```

## 半年总结评分

**在月度评分的基础上，加入时间维度**

1. 画出6个月的分数变化曲线
2. 找出趋势：哪些维度在持续进步？哪些在原地踏步？哪些在退步？
3. 找模式：什么时候开始出现突破？突破的触发事件是什么？
4. 给震撼的结论：不是"你进步了"，而是"你和半年前比，最大的变化是X，这是因为你做了Y"

**输出格式：**
```
## 半年总结

### 分数变化总览
[维度]：[M1]→[M2]→[M3]→[M4]→[M5]→[M6]

### 重大突破
维度：...
触发事件：...
现在状态：...

### 持续退步
维度：...
原因分析：...
如果不改变会怎样：...

### 核心模式变化
发现：...
证据：...

### 我的判断
[说一个会让他震惊的结论]
```

## 自检规则

每次评分后，自检：

1. **证据完整性** — 每个维度都有2条以上证据吗？
2. **评分一致性** — 这个分比上月高/低，理由充分吗？
3. **可反驳性** — 如果用户反对，我能拿出证据吗？
4. **优先级清晰** — 最需要突破的维度明确吗？

如果任何一个没通过，重新检查。
