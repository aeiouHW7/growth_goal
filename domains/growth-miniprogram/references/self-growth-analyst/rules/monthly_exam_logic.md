# 月考生成逻辑

## 触发时间
每月最后一天 20:00 (Asia/Shanghai)

---

## 数据汇总

1. **读取当月所有碎碎念**：`memories/daily_raw/YYYY-MM-*.md`
2. **读取当月所有周考记录**：`memories/weekly_summaries/YYYY-WW.md`
3. **读取上月评分**：`config/capability_baseline.json`
4. **查询 retrieval_index.json**：patterns + pending_tracking

---

## 月报结构

### 1. 本月概述
- 本月碎碎念数量
- 本月主要话题（出现最多的3个关键词）
- 本月行为模式变化

### 2. 20维评分报告

对每个维度输出：
```markdown
### [维度名]：上月分 → 本月分 (变化方向)

**证据1（来源：MM-DD碎碎念）**
> [引用内容]
> 分析：这句话/行为为什么说明这个能力的变化

**证据2（来源：WW周周考）**
> [引用内容]
> 分析：...

**AI观察**
> 变化结论

[待验证]（若证据不足）
```

### 3. 评分变化规则
| 变化 | 条件 |
|------|------|
| +0.5 | 1条进步证据 + 无倒退 |
| +1 | 2条不同场景进步证据 |
| -0.5 | 1条倒退证据或2次同样错误 |
| 不变 | 证据不足或正负抵消 |

### 4. 最低3维度重点分析
```
1. [最低分维度]
   问题：...
   下月建议：...

2. [次低]
   ...

3. [第三低]
   ...
```

### 5. probe进化记录（本月）
- 本月触发了哪些probe进化？原因？
- 进化后的变化是什么？

### 6. 下月提升重点
- 最需要突破的1个维度
- 最容易见效果的1个行动
- 需要用户特别关注的1个模式

---

## 月考评分前置检查

每次评分前自检：
- [ ] 每个维度有2条以上证据？
- [ ] 变化理由充分？
- [ ] 若用户反对，我能拿出证据？
- [ ] 最需要突破的维度明确？

---

## 月底最后一天操作

1. 生成月报内容
2. 更新 `config/capability_baseline.json`（history追加，不覆盖）
3. 写入 `memories/monthly_reports/YYYY-MM.md`
4. 通过 message tool 发送月报到用户 Feishu
5. 若某些维度证据不足 → 标注 `[待验证]`，月报注明"建议下月补充碎碎念来验证"
