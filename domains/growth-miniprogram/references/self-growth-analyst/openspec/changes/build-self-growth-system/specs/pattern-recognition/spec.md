# pattern-recognition

## ADDED Requirements

### Requirement: 三类模式触发器必须明确定义

**频率触发**：
- 同一关键词（如"拖延"、"后悔"、"需求变了"）在7天内出现在≥3条碎碎念中
- 系统 SHALL 识别重复的词、话题或情绪
- 触发后必须在碎碎念记录里标记 `[模式触发]`
- 必须追问，不能只记录

#### Scenario: 频率触发判定
- **WHEN** 同一关键词在7天内出现第3次
- **THEN** 系统生成追问："你不是第一次说这个。到底是怎么回事？"
- **AND** 在记录中标记 `[模式触发]`
- **AND** 在 `memories/retrieval_index.json` 中更新 patterns

**矛盾触发**：
- 用户在某次考核中说"我 A 能力很强"但在日常记录中有≥2次与 A 相反的行为
- 系统 SHALL 指出"A"和"B"的矛盾，问用户怎么想

#### Scenario: 矛盾触发判定
- **WHEN** 用户说"我决策没问题"但实际提到两次"后悔选了A方案"
- **THEN** 系统输出："你刚说决策没问题，但你提到了两次'后悔选了A'。这两件事放一起看，你对'决策没问题'的定义可能和真实情况不符。"
- **AND** 记录矛盾点

**静默触发**：
- 同一维度连续2次月考分数不变或下降，且无任何改进证据
- 用户说"我要提升X"但3周内没有任何相关新行为

#### Scenario: 静默触发判定
- **WHEN** 政治智慧连续两次月考都是4分
- **THEN** 系统输出："你在政治智慧上连续两次没有进步。不是分数没变——是你没有任何新行为。你到底是在'维持现状'还是'卡住了不敢动'？"

### Requirement: 挑战模式 vs 记录模式切换

进入挑战模式的条件（满足任一）：
1. 频率触发命中
2. 矛盾触发命中
3. 静默触发命中
4. 用户主动问"你觉得我有什么问题"

挑战模式输出规则：
- 必须有证据（引用用户之前说过的具体内容）
- 必须指出用户没意识到的东西
- 不能只是"你说得对"
- 每次只追问一个问题

记录模式：
- 不满足挑战模式触发条件的碎碎念
- 只记录，不回复
- 除非用户主动说"你评评"或"你怎么看"

#### Scenario: 模式切换
- **WHEN** 用户碎碎念内容触发频率触发
- **THEN** 系统进入挑战模式，输出追问
- **WHEN** 用户碎碎念内容不满足任何触发条件
- **THEN** 系统只记录到当日文件，不回复用户

### Requirement: 不触发的情况明确化

以下情况 SHALL 不触发（只记录）：
- 用户只是情绪宣泄，没有具体内容
- 用户在分享好消息
- 用户明确说"我就是随便说说"
- 用户在说别人的事不是在说自己
- 连续两天提到同一件事但背景不同

#### Scenario: 假阳性过滤
- **WHEN** 用户连续两天提到同一件事但背景不同
- **THEN** 系统判断：这不是模式，可能是正常不同场景
- **AND** 只记录，降低权重
- **WHEN** 用户说了一个词但没展开
- **THEN** 不算一次触发，等他展开再计数
