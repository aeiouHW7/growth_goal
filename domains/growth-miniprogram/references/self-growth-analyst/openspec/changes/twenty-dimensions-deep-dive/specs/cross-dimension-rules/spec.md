## ADDED Requirements

### Requirement: 跨维度关联规则库 (Cross-Dimension Rules Repository)

作为已验证的跨维度行为关联的永久记录库，供分析引擎和 Coach Agent 在决策时参考。本规则库由 `pattern-miner` 创建并维护，经过验证的规则永久存入此处供查询。

---

#### Scenario: 规则库查询——行为触发

- **WHEN** 系统需要输出某项行为分析的洞察时
- **THEN** 分析引擎查询本规则库，查找是否有已记录的规则与当前行为相关联；若存在匹配规则，在洞察输出中标注：`触发规则：[规则内容]（证据强度: ★★★）`

---

#### Scenario: 规则结构定义

- **WHEN** 创建或更新跨维度规则时
- **THEN** 每条规则必须包含以下字段：
  - **规则内容**：格式为"前提行为 → 预期结果"，行为和结果均为具体维度标签
  - **证据强度**：1-5星（★ = 初步观察，★★ = 重复观察，★★★ = 多场景验证，★★★★ = 跨用户验证，★★★★★ = 长期稳定验证）
  - **最后验证日期**：格式 `YYYY-MM-DD`，每次触发后自动更新
  - **失效条件**：描述在什么情况下该规则不再成立（如"前提行为中断超过30天"或"用户处于重大生活变化期间"）
  - **来源**：指向具体观察记录或分析报告的引用

---

#### Scenario: 规则示例——睡眠与执行力

- **WHEN** 创建"睡眠 → 执行力"关联规则
- **THEN** 规则条目示例：
```
[规则] 睡眠质量提升 → 执行力提升
证据强度: ★★★
最后验证: 2026-04-20
失效条件: 用户处于高压工作期或健康状态重大变化期间
来源: pattern-miner 观察记录（2026-03-15, 2026-03-22, 2026-04-05）
```

---

#### Scenario: 规则更新与降级

- **WHEN** 某条规则在超过90天未被触发，或用户行为模式发生根本性变化
- **THEN** Analyst Agent 评估该规则是否降级或标注为"待验证"；降级规则保留在库中但证据强度降低，并在规则条目中标注"最后触发：YYYY-MM-DD，已失效"

---

#### Scenario: Coach Agent 规则调用

- **WHEN** Coach Agent 需要设计干预策略时
- **THEN** 可查询本规则库获取当前有效规则列表，基于证据强度选择高置信度规则作为干预设计的参考；若规则证据强度低于★★，建议Coach Agent采取保守策略，不将规则预测作为确定性的干预依据
