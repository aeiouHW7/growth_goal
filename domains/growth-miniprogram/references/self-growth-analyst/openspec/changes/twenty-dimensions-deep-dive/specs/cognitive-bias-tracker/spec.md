## ADDED Requirements

### Requirement: 固有认知偏误追踪器 (Cognitive Bias Tracker)

与 `cognitive_bias_log.md`（日常偏误日志）不同，本追踪器专注于长期固化的、反复出现的认知偏误模式。当同一偏误在日志中出现达到指定阈值时，将其标记为"固有偏误"并纳入追踪器管理，优先于一般偏误进行处理。

---

#### Scenario: 偏误升级判定——5次阈值

- **WHEN** 同一认知偏误（如"全或无思维""灾难化""贴标签"等）在 `cognitive_bias_log.md` 中出现5次或以上
- **THEN** 该偏误被标记为"固有偏误"，从日志移入 `cognitive_bias_tracker.md`，记录格式：
```
{
  偏误名称: <偏误类型>,
  首次出现: <日期>,
  出现次数: <总次数>,
  触发情境: <最常见的触发场景>,
  典型干预: <Coach Agent建议干预方式>
}
```

---

#### Scenario: 固有偏误优先级高于普通偏误

- **WHEN** 在同一次分析中同时识别出固有偏误和普通偏误
- **THEN** Analyst Agent 和 Coach Agent 必须优先处理固有偏误，在洞察和干预建议中先点出固有偏误的名称和模式，帮助用户建立认知意识，再处理普通偏误

---

#### Scenario: Coach Agent 固有偏误干预

- **WHEN** Coach Agent 进行干预时识别到用户正处于固有偏误模式
- **THEN** Coach Agent 明确说出该偏误的名称（如"你现在正在使用全或无思维——把所有事分成成功或失败，没有中间地带"），使用心理教育式的干预语言，帮助用户建立元认知能力，而非仅给出安慰或建议

---

#### Scenario: 固有偏误数据积累与回顾

- **WHEN** 每90天进行一次认知偏误回顾
- **THEN** 分析引擎检查 `cognitive_bias_tracker.md` 中各固有偏误的出现频率变化；若某固有偏误在过去90天内未再触发，评估是否降级为普通偏误（保留追踪但降低优先级）；若出现新模式，则新增条目

---

#### Scenario: 固有偏误干预效果标注

- **WHEN** Coach Agent 对某固有偏误进行干预后，用户在下一次对话中表现出偏误减轻的迹象
- **THEN** 在该固有偏误条目下记录"干预有效"标记和日期，供后续参考；若连续3次干预均有效，考虑更新该偏误的典型干预建议
