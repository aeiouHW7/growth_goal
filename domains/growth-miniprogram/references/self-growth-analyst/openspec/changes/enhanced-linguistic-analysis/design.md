## Context

用户需要一个"世界级全方位监控导师"，不仅仅是简单的模式匹配。当前系统的问题是：分析太浅，只看"说了什么"，不看"怎么说"和"为什么这么说"。

## Decisions

### D1: 增强 linguistic_analyzer 作为主要入口

**决策**: 扩展 linguistic_analyzer 的输出，不只是归因/情绪/防卫，还要包含语言风格分析。

**原因**:
- 用户自己发现"表达不清晰"（频率3），说明语言风格本身就是分析对象
- "放纵"和"失控"同样行为不同语气，暗示不同心理状态
- 语气分析可以捕捉防御机制

**扩展字段**:
```json
{
  "tone_markers": ["justification"|"deflection"|"neutralization"|"hot"|"cool"],
  "defense_mechanisms": ["rationalization"|"minimization"|"blame_shift"],
  "emotional_temperature": "hot(>7)|warm(4-7)|cool(<4)",
  "implicit_intent": "推断出的没说出来的意图"
}
```

### D2: 新增 time_pattern_analyzer 作为L2引擎

**决策**: 基于消息时间推断用户状态，作为独立引擎。

**原因**:
- 凌晨发消息→透支/亢奋/逃避
- 早上发消息→刚睡醒/精力尚可
- 深夜发消息→失眠/思考/内耗
- 周末发消息→更放纵/更真实

**触发时机**: 不是每次碎碎念都跑，是当消息时间异常时触发（L2）。

### D3: 防御机制检测器内嵌到 linguistic_analyzer

**决策**: 不单独建引擎，把防御机制检测集成到 linguistic_analyzer 里。

**原因**:
- 防御机制是语言风格的子集
- 独立引擎会增加上下文负担
- 集成更高效

**检测模式**:
| 防御机制 | 关键词 | 扣分 |
|---------|-------|------|
| 合理化 | "因为"/"所以"/"这样就能" | -0.5 |
| 最小化 | "一点点"/"还行"/"没什么" | -0.5 |
| 转移话题 | "算了"/"不说这个了"/"反正" | -0.5 |
| 借口 | "周六可以"/"工作太忙" | -0.5 |

### D4: 信号厚度评分扩展到8维度

**决策**: 在原5维度基础上新增3个维度。

| 新增维度 | 权重 | 说明 |
|---------|:--:|------|
| 隐层意图 | 2 | 能推断出没说出来的意图 |
| 防御机制分 | -1 | 有防御机制时总分折算 |
| 语言温度 | 1 | 冷淡=低参与，热情=高参与 |

**总分公式**: `(sum of positive scores) * (1 + defense_penalty)`

### D5: 7步流程强制执行

**决策**: 在 SKILL.md 里明确每步必须有输出，空输出也算输出。

**原因**:
- 当前执行中跳步骤导致分析不稳定
- 用户要求"强制规范"
- 每步的输出是下一步的输入，跳过会导致链路断裂

## Risks / Trade-offs

[风险] 增加的分析维度会增加token消耗 → 缓解：L2引擎按需触发，不每次都跑

[风险] 隐层意图可能推断错误 → 缓解：只输出推断，不输出结论，给用户校验机会

[风险] 防御机制扣分可能让用户觉得被批评 → 缓解：输出的是"检测到X"，不是"你在找借口"
