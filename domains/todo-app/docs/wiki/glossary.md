# TODO App - 业务术语表

## 核心概念

### Todo（待办事项）

**定义**: 用户创建的单个任务或待办项。

**属性**:
- **ID**: 唯一标识符
- **标题（Title）**: 待办事项的简短描述
- **完成状态（Completed）**: 布尔值，表示是否已完成
- **创建时间（Created At）**: 待办事项的创建时间戳

**业务规则**:
- 标题不能为空
- 标题长度限制为 1-200 字符
- 创建时默认为未完成状态
- 完成状态可以在创建后修改

---

### Todo List（待办列表）

**定义**: 用户的所有待办事项的集合。

**特性**:
- 支持查看所有待办事项
- 支持按状态筛选（全部/未完成/已完成）
- 按创建时间倒序排列（最新的在前）

---

## 业务流程

### 创建待办事项

```
用户输入标题 → 验证输入 → 创建 Todo → 添加到列表 → 显示成功
```

**验证规则**:
1. 标题不能为空
2. 标题长度 1-200 字符
3. 如果验证失败，显示错误信息

---

### 切换完成状态

```
用户点击 Todo → 切换完成状态 → 更新数据库 → 刷新列表
```

**行为**:
- 未完成 → 完成：显示删除线样式
- 完成 → 未完成：恢复正常样式

---

### 删除待办事项

```
用户点击删除按钮 → 确认（可选）→ 删除 Todo → 从列表移除
```

**注意**:
- 删除操作不可逆
- 当前版本无需二次确认（简化）

---

## 业务约束

### MVP 范围（v0.1.0）

**包含功能**:
- ✅ 创建待办事项
- ✅ 查看待办列表
- ✅ 切换完成状态
- ✅ 删除待办事项

**不包含功能**（未来版本）:
- ❌ 用户认证
- ❌ 多用户隔离
- ❌ 待办分类/标签
- ❌ 截止日期
- ❌ 优先级
- ❌ 附件上传

---

## 用户角色

### MVP 阶段

**单用户模式**: 
- 无需登录
- 所有待办事项共享
- 适用于个人使用或演示

### 未来版本

**多用户模式**:
- 需要注册/登录
- 每个用户独立的待办列表
- 数据隔离

## 常见问题

**Q: 待办事项有数量限制吗？**

A: MVP 版本无限制。生产环境建议每用户限制 1000 条。

**Q: 删除的待办事项可以恢复吗？**

A: 当前版本不支持。未来可考虑"软删除"功能。

**Q: 支持批量操作吗？**

A: 当前版本不支持。MVP 专注单个操作的实现。
