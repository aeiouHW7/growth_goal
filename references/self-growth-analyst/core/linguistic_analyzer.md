# 语言学分析引擎 (Linguistic Analyzer)

## 核心能力
当用户输入文本时，提取以下结构化数据：

### 归因模式分析
- 外部归因词：怪、运气、外部环境、别人让、没办法
- 内部归因词：我应该、我可以、我选择、我的问题
- 合理化词：其实、就是、也算、反正

### 情绪词频统计
统计以下情绪词出现次数：焦虑/愤怒/沮丧/兴奋/疲惫/逃避/防御
计算情绪净值（正面-负面）

### 防卫信号检测
- 否认词：没有、我不是、我没...
- 转移词：但是、其实、算了
- 自我保护：都怪、都因为

## 调用时机
每次碎碎念处理时，Observer 调用此引擎提取数据。

## 输出格式
```json
{
  "attribution": "external|internal|rationalization",
  "emotion_scores": {"positive": N, "negative": N, "dominant": "xxx"},
  "defense_signals": ["xxx", "xxx"],
  "linguistic_flags": ["xxx"]
}
```

## 运行指令

分析用户输入文本，输出以下维度的结构化数据：

### 分析步骤

**Step 1: 归因模式判定**
扫描文本，统计以下关键词出现情况：
- 外部归因词：「怪」「运气」「环境」「别人让」「没办法」「都怪」「都因为」
- 内部归因词：「我应该」「我可以」「我选择」「我的问题」「是我」
- 合理化词：「其实」「就是」「也算」「反正」「算了」「不过」

判定规则：
- 外部词 > 内部词 × 2 → attribution = "external"
- 内部词 > 外部词 × 2 → attribution = "internal"
- 合理化词出现 ≥ 2次，或内部词≈外部词时 → attribution = "rationalization"
- 默认 → attribution = "rationalization"（默认假设防御性叙事）

**Step 2: 情绪词频统计**
扫描以下情绪词，记录出现次数（大小写不敏感，词根匹配）：

| 情绪类型 | 关键词列表 |
|---------|-----------|
| 焦虑类 | 焦虑、不安、担心、紧张、慌、心慌、忐忑、压力、焦虑 |
| 愤怒类 | 生气、愤怒、恼火、烦躁、烦、讨厌、恨、气 |
| 沮丧类 | 沮丧、失落、挫败、无力、绝望、灰心、丧气、泄气 |
| 兴奋类 | 兴奋、激动、开心、快乐、爽、激动、兴奋 |
| 疲惫类 | 累、疲惫、疲劳、困、倦、乏力、没精神、身心俱疲 |
| 逃避类 | 逃避、拖延、算了、不想管、不想、回避、躲 |
| 防御类 | 不可能、没道理、我不是、没有、谁说、凭什么 |

计算：
- positive_score = 兴奋类次数
- negative_score = 焦虑类 + 愤怒类 + 沮丧类 + 疲惫类 + 逃避类
- dominant_emotion = 得分最高的情绪类型名称

**Step 3: 防卫信号检测**
检测以下防卫机制信号：

1. **否认信号**：出现「没有」「我不是」「我没」「别说我」「谁说的」等否认词
2. **转移信号**：出现「但是」「其实（后文转折）」「算了（放弃讨论）」等转移词
3. **自我保护信号**：出现「都怪」「都因为」「都是他」「都怪环境」等外部归因词
4. **合理化信号**：出现「其实（自我辩护）」「反正」「也算」等合理化词

记录检测到的所有防卫信号类型和具体引文。

**Step 4: 标记语言标志**
生成以下标志：
- `over_generalization`：过度概括（出现「总是」「从不」「所有人」「没人」）
- `victim_language`：受害者语言（出现「被迫」「没办法」「不得不」）
- `external_locus`：外部控制点语言
- `dichotomous_thinking`：非黑即白思维（出现「要么...要么...」「不是...就是...」）

### 最终输出
输出 JSON 格式结果，包含完整分析过程和引用证据。

## 语言风格分析

### 输出字段

```json
{
  "tone_markers": {
    "justification": ["因为...所以", "为了", "这样就能"],
    "deflection": ["算了", "不说这个了", "反正", "就这样"],
    "neutralization": ["也没什么", "就这样吧"],
    "minimization": ["一点点", "还行", "没什么"]
  },
  "defense_mechanisms": ["rationalization", "minimization", "blame_shift", "denial"],
  "emotional_temperature": 0-10,
  "implicit_intent": "推断出的隐层意图（没说出来的）",
  "action_verbs_count": 0,
  "state_verbs_count": 0
}
```

#### tone_markers 语气标记数组

| 类型 | 关键词 | 说明 |
|------|--------|------|
| justification | "因为...所以"、"为了"、"这样就能" | 论证性语气，为行为找理由 |
| deflection | "算了"、"不说这个了"、"反正"、"就这样" | 回避性语气，试图转移或终止话题 |
| neutralization | "也没什么"、"就这样吧" | 中和性语气，降低重要性 |
| minimization | "一点点"、"还行"、"没什么" | 最小化语气，轻描淡写 |

#### defense_mechanisms 防御机制数组

| 类型 | 说明 |
|------|------|
| rationalization | 合理化：用看似合理的理由解释不合理的行为 |
| minimization | 最小化：故意缩小问题或情绪的严重程度 |
| blame_shift | 外部归因：把责任推给外部环境或他人 |
| denial | 否认：拒绝承认某种情绪或行为 |

#### emotional_temperature 情感温度 (0-10)

| 区间 | 标签 | 说明 |
|------|------|------|
| > 7 | hot | 高参与/情绪化，表达强烈 |
| 4-7 | warm | 正常，平和参与 |
| < 4 | cool | 冷淡/防御，回避深入 |

#### implicit_intent 隐层意图

推断用户没有直接说出来的真实意图。

**隐层意图推断示例：**

| 文本 | 推断意图 |
|------|---------|
| "周六可以放纵" | 为失控行为寻找道德许可 |
| "工作太累了" | 为放弃找外部归因 |
| "就是烦" | 回避具体描述，模糊情绪 |
| "随便吧" | 隐藏失望或无奈 |
| "我知道该怎么做" | 可能是在压制焦虑的防御性表态 |

#### action_verbs_count 行动词数量

统计表示动作行为的动词出现次数（想要、打算、决定、开始、去做 等）。
行动词多 → 叙述倾向主动；行动词少 → 可能处于停滞或观望状态。

#### state_verbs_count 状态词数量

统计表示心理或情感状态的动词出现次数（觉得、感觉、好像、应该、想 等）。
状态词多 → 叙述倾向内省；状态词少 → 可能更关注外部事件。

---

### 防御机制检测规则

扫描文本关键词，根据以下规则计算 emotional_temperature 基准分（满分10分），并按防御机制扣分：

| 防御机制 | 关键词示例 | 扣分 |
|---------|-----------|------|
| rationalization | "因为"、"所以"、"这样就能"、"为了" | -0.5 |
| minimization | "一点点"、"还行"、"没什么"、"就一点" | -0.5 |
| blame_shift | "是他让我"、"没办法"、"不得不" | -0.5 |
| deflection | "算了"、"不说这个了"、"反正"、"就这样" | -0.5 |

**初始基准分计算：**
- 情绪词（焦虑/愤怒/沮丧/疲惫/逃避）每出现一次 → +1.5 分（情绪强度）
- 兴奋词每出现一次 → +1.0 分
- 防御机制关键词每触发一项 → 扣 0.5 分
- 最终分数限制在 0-10 范围内

## Coach 联动说明
将输出结果传递给 Coach，Coach 根据归因模式和防卫信号类型，选择合适的干预策略：
- attribution="external" + defense_signals 丰富 → 用户可能陷入受害者心态
- attribution="rationalization" → 用户可能在合理化逃避行为
- 情绪净值显著为负 → 优先处理情绪，再谈行动
