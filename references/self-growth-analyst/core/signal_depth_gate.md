# Signal Depth Gate — 信号厚度判定引擎 v2

## 引擎类型
L1 常驻引擎 — 每次碎碎念必加载，约300 tokens。

## 触发时机
linguistic_analyzer 输出后立即执行，先于 auto_insight_generator。

---

## 信号厚度评分（8维度）

| 维度 | 权重 | 评分标准 |
|------|:--:|------|
| 具体行为 | 3 | "我做了X"=3，"有人搞我"=1，完全无=0 |
| 归因指向 | 2 | "因为X"=2，"不知道"=0 |
| 情绪具体 | 2 | "焦虑担心OKR"=2，"烦"=0.5 |
| 时间锚定 | 2 | "今天下午"=2，无时间=0 |
| 涉及维度 | 1 | 提到能力维度=1，未提及=0 |
| 隐层意图 | 2 | 能推断出没说出来的=2，不能=0 |
| 防御机制 | -1 | 有防御机制(每种)扣0.5，上限-2 |
| 语言温度 | 1 | hot=1，warm=0.5，cool=0 |

### 综合评分公式

```
base_score = sum(维度得分)
defense_penalty = sum(0.5 for 每种防御机制)
final_score = base_score - defense_penalty
```

### 判定规则

| 分数区间 | 判定 | 动作 |
|---------|------|------|
| ≥8 | 极厚信号 | 直接深度分析，跳过追问 |
| 5-7 | 正常信号 | 继续 Step 4-6 |
| 3-4 | 薄信号 | 缺口探测≤2次 |
| <3 | 极薄信号 | 追问1次，无改善→归档[极薄] |

### 计算示例

用户说"因为周六可以放纵，所以凌晨4:30睡"：
- 具体行为=3 + 归因指向=1 + 情绪具体=0.5 + 时间锚定=2 + 涉及维度=0 + 隐层意图=1 + 语言温度=0.5 = 8
- 防御机制扣分：rationalization(0.5) + deflection(0.5) = 1
- final_score = 8-1 = 7
→ 7分，正常信号

## 缺口类型与追问方向

| 缺口类型 | 触发条件 | 追问方向 |
|---------|---------|---------|
| [行为缺失] | 具体行为 = 0 | 「你今天具体做了什么？」 |
| [归因缺失] | 归因指向 = 0 | 「你觉得是什么原因造成的？」 |
| [情绪模糊] | 情绪具体 < 1 | 「你说的XX，具体是哪种感受？焦虑/无力/愤怒？」 |
| [时间缺失] | 时间锚定 = 0 | 「这是什么时候的事？」 |
| [隐层缺失] | 隐层意图 = 0 | 「你内心真正想要的是什么？」 |
| [多维度缺失] | 多个=0 | 只追问得分最高的那个缺口 |

---

## 锚定追问规则

追问必须包含以下至少一项：
1. retrieval_index 中频率 ≥2 的模式引用
2. active_conflicts 中的相关矛盾
3. pending_actions 中状态为 pending 的行动

**禁止空泛追问**：❌「你再多说说」❌「还有什么？」❌「详细说说？」

---

## 追问上限

- **极厚信号（≥8）**：跳过追问，直接深度分析
- **正常信号（5-7）**：无需追问，继续 Step 4-6
- **薄信号（3-4）**：缺口探测≤2次
- **极薄信号（<3）**：追问1次，无改善→归档

### 第1次追问
- 根据缺口类型生成锚定追问

### 第2次追问（薄信号专属）
- 用户补充后重新评分，仍在 3-4 区间
- 换一个缺口方向追问

---

## [极薄输入] 归档格式

```markdown
[极薄输入] 信号厚度评分: N/10
缺口: [行为缺失]/[归因缺失]/[情绪模糊]/[时间缺失]/[隐层缺失]
追问次数: 1
状态: 已归档，未分析
```

## 判定流程

1. 对 linguistic_analyzer 输出做信号厚度评分（8维度）
2. 计算 final_score = base_score - defense_penalty
3. ≥8 → 极厚信号，直接深度分析，跳过追问
4. 5-7 → 正常信号，继续 Step 4-6
5. 3-4 → 薄信号，缺口探测≤2次
6. <3 → 极薄信号，追问1次，无改善→归档[极薄输入]，NO_REPLY
