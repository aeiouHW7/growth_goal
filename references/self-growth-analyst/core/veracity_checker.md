# 真实性检测器 (Veracity Checker)

## 核心能力
检测用户陈述中的自我欺骗和认知盲点。

## 检测类型

### 1. 自我美化
用户把外部成功归因自己，失败归因外部。
检测词："都是我的功劳"、"运气不好"

### 2. 计划谬误
用户高估自己完成任务的效率。
检测：对比历史数据和实际完成率。

### 3. 确认偏误
用户只收集支持自己观点的信息。
检测：检查是否有"虽然...但是..."结构。

### 4. 盲点检测
用户不知道自己不知道什么。
检测：当用户说"我知道"时，检查是否在某个领域确实知道。

## 输出格式
```json
{
  "deception_type": "self_flattery|planning_fallacy|confirmation_bias|blind_spot",
  "evidence": "具体引用",
  "blind_spot_index": 1-10,
  "coaching_prompt": "直接指出问题的prompt"
}
```

## 运行指令

### 检测类型1: 自我美化 (Self-Flattery)

**检测逻辑：**
扫描用户陈述，识别以下模式：

**成功归因外部（贬低自己努力）：**
- 「就是运气好」「纯属侥幸」「碰巧」
- 「正好赶上」「时机好」「别人帮的」
- 成功时：说是运气/环境/他人；失败时：说是我能力不行

**失败归因外部（推卸责任）：**
- 「都怪环境」「都是别人的问题」「运气不好」
- 「市场不好」「同事不配合」「领导不支持」
- 失败时：说是外部因素；成功时：说是自己厉害

**检测规则：**
1. 如果用户成功谈成功时出现外部归因词 ≥2次 → 检测到 self_flattery
2. 如果用户谈失败时出现外部归因词，且成功后出现内部归因词 → 检测到 self_flattery
3. 如果成功时用「运气」「碰巧」「正好」，失败时用「我不行」「我能力不够」 → 严重 self_flattery

### 检测类型2: 计划谬误 (Planning Fallacy)

**检测逻辑：**
对比用户历史承诺与实际完成：

**收集用户历史数据：**
- 用户过去说「我会...」「下周开始...」「一定...」的记录
- 对比实际完成情况

**检测规则：**
1. 如果用户过去3次承诺中有2次以上未完成 → 检测到 planning_fallacy
2. 如果用户对任务完成时间的估计总是过于乐观 → 检测到 planning_fallacy
3. 如果用户说「只需要...」「很快就能...」「很简单」但实际很复杂 → 检测到 planning_fallacy

**计算计划谬误指数：**
```
计划谬误指数 = 历史承诺完成率
- 如果完成率 < 50% → 高风险 planning_fallacy
- 如果完成率 50-70% → 中等风险
- 如果完成率 > 70% → 低风险
```

### 检测类型3: 确认偏误 (Confirmation Bias)

**检测逻辑：**
扫描用户陈述中的结构：

**非黑即白结构：**
- 「虽然（某事实）但是（我的观点）」→ 用但是否定事实
- 「我承认...但是...」→ 但是后面的才是真实立场
- 「你说得对...但是...」→ 表面同意，实际坚持己见

**选择性收集信息：**
- 只提支持自己观点的证据
- 忽略或否认反对证据
- 对批评意见的本能反驳

**检测规则：**
1. 如果出现「虽然...但是...」结构，且但是后面的内容否定了前面的事实 → 检测到 confirmation_bias
2. 如果用户只提正面证据，不提反面证据 → 检测到 confirmation_bias
3. 如果用户对反对意见有即时的、情绪化的反驳 → 检测到 confirmation_bias

### 检测类型4: 盲点检测 (Blind Spot)

**检测逻辑：**
识别用户「我知道」的领域是否真的知道：

**检测关键词：**
- 「我懂」「我知道」「我明白」「我了解」
- 「这个很简单」「没什么难的」「一看就会」

**检测规则：**
1. 如果用户说「我懂AI」但不懂基本原理 → 盲点
2. 如果用户说「我知道怎么做」但从未做过 → 盲点
3. 如果用户说「很简单」但实际上有复杂度 → 盲点

**评估盲点指数（1-10）：**
- 1-3分：用户有适度自知之明
- 4-6分：存在一些盲点，但有基本判断
- 7-10分：严重高估自己，盲点很多

**评估方法：**
- 观察用户是否经常说「我知道」但实际不知道
- 对比用户声称懂的东西和实际表现
- 检查用户是否经常被事实打脸

### 输出格式

```json
{
  "deception_type": "self_flattery|planning_fallacy|confirmation_bias|blind_spot",
  "evidence": "具体引用原文",
  "confidence": 0.0-1.0,
  "severity": "low|medium|high",
  "blind_spot_index": 1-10,
  "coaching_prompt": "直接指出问题的prompt",
  "reflection_questions": ["帮助用户自我反思的问题列表"]
}
```

### Coach 联动说明

**重要原则：**
- 不要直接说「你在骗自己」
- 用具体证据引导用户自己发现问题
- 保持好奇而非评判的态度

**Coaching prompt 示例：**
```
# self_flattery:
"你把这次的成功归因于运气。让我好奇的是——在这个过程中，有什么是只有你做到而别人做不到的？"

# planning_fallacy:
"你说这周能完成。现在是周末了，回顾一下，实际发生了什么？我们来对比一下当初的估计和实际情况。"

# confirmation_bias:
"你提到了这些支持你观点的证据。让我好奇的是——有没有什么反面的证据？什么情况下你的观点可能不成立？"

# blind_spot:
"你说这个领域你已经掌握了。让我测一下——如果让你解释一下这个概念的核心原理，你会怎么说？"
```

## 调用时机
- 每次用户有重大成功/失败时调用
- 每次周/月考时调用
- 发现与capability_baseline差距大时调用
- 当用户说「我知道」「很简单」「没问题」等时触发盲点检测
