# 习惯行为引擎 (Habit Behavior Engine)

## 福格行为模型 B = M × A × P
- B (Behavior): 行为
- M (Motivation): 动机
- A (Ability): 能力
- P (Prompt): 提示

## 诊断流程
当用户说"知道应该做但没做"时：

### Step 1: 判断哪个不足
- M不足信号："没动力"、"不想做"、"没意义"
- A不足信号："太难了"、"不会做"、"不知道怎么做"
- P不足信号："忘了"、"没想起来"、"没时间"

### Step 2: 诊断结论
输出：
```json
{
  "dimension": "xxx",
  "fogg_diagnosis": "motivation|ability|prompt",
  "evidence": ["具体证据"],
  "recommendation": "xxx"
}
```

### Step 3: Coach联动
根据诊断结果，Coach生成针对性干预：
- M不足 → 连接目标（为什么做这件事对你重要）
- A不足 → 降低难度（最小可执行步骤是什么）
- P不足 → 增加提醒（什么可以触发你开始）

## 运行指令

### 福格模型诊断

**当用户描述"知道应该做但没做"的场景时，执行以下诊断：**

### Step 1: 收集行为描述

识别用户描述的目标行为，例如：
- "知道要运动但没去"
- "想写公众号但一直拖延"
- "知道要学习但刷手机"
- "应该给客户打电话但逃避"

### Step 2: 检测三维度信号

**Motivation（动机）不足信号：**
扫描用户描述，检测以下关键词/模式：
- 「没动力」「不想」「没意义」「不值得」「有什么意思」「懒得」「不想动」
- 「反正」「也没用」「做了又怎样」「没价值」
- 用户表达了结果不重要/不吸引人

**Ability（能力）不足信号：**
扫描用户描述，检测以下关键词/模式：
- 「太难了」「不会」「不知道怎么」「不知道怎么开始」
- 「太复杂」「太麻烦」「步骤太多」「不知道怎么做」
- 「没时间」「没条件」「没资源」「环境不允许」
- 用户提到了具体执行障碍

**Prompt（提示）不足信号：**
扫描用户描述，检测以下关键词/模式：
- 「忘了」「没想起来」「一忙就忘」「总是忘」
- 「没提醒」「没人催」「没有触发」「没有习惯」
- 「时机不对」「没到那个点」「没养成习惯」

### Step 3: 判定主维度

判定规则：
1. 如果 A 不足信号最强（提到具体执行障碍/太难/不会）→ primary = "ability"
2. 如果 M 不足信号最强（提到没意义/不想/懒得）→ primary = "motivation"
3. 如果 P 不足信号最强（提到忘了/没提醒/没触发）→ primary = "prompt"
4. 如果多个维度都有信号，按以下优先级：ability > motivation > prompt

### Step 4: 诊断输出

```json
{
  "target_behavior": "用户想做的行为",
  "fogg_diagnosis": {
    "motivation_score": 0-10,
    "ability_score": 0-10,
    "prompt_score": 0-10,
    "primary_deficiency": "motivation|ability|prompt",
    "deficiency_evidence": ["具体引用"]
  },
  "tiny_habit_suggestion": "如果诊断为ability不足，给出最小可行步骤",
  "motivation_connection": "如果诊断为motivation不足，给出目标连接问题",
  "prompt_design": "如果诊断为prompt不足，给出触发设计建议"
}
```

### Step 5: Coach 联动干预

**当 diagnosis.primary_deficiency = "motivation" 时：**
生成以下类型的问题让 Coach 询问用户：
- "为什么这件事对你重要？"
- "如果做到了，你会是什么样的状态？"
- "这件事和你的什么目标相关？"
- "一年后你会后悔没做这件事吗？"

**当 diagnosis.primary_deficiency = "ability" 时：**
生成以下类型的问题让 Coach 询问用户：
- "最小可行的第一步是什么？"
- "能不能把这件事拆到5分钟就能完成？"
- "有什么方法可以让它变简单？"
- "如果只能做一件事，是什么？"

**当 diagnosis.primary_deficiency = "prompt" 时：**
生成以下类型的问题让 Coach 询问用户：
- "什么时间/地点最适合做这件事？"
- "有什么现有的习惯可以绑定？"
- "谁能提醒你？"
- "什么信号可以触发你开始？"

### 福格模型核心原则

**动机和能力可以互相替代：**
- 高动机可以弥补高难度（愿意做难的事）
- 高能力可以弥补低动机（容易做到就愿意做）
- 当动机不高且能力不足时，行为不会发生

**提示必须存在才能触发行为：**
- 没有提示，即使高动机+高能力，行为也不会发生
- 提示可以是对比（看到跑步鞋）、时间（每天8点）、事件（前一个习惯）

**降低难度比提高动机更有效：**
- 把行为缩小到微小版本，比说服自己努力更容易
- 「做一个俯卧撑」比「运动30分钟」更可能执行

## Coach 联动说明

诊断结果传递给 Coach 后，Coach 生成针对性干预方案：
- 不要同时解决所有维度，聚焦 primary_deficiency
- 如果是 ability 不足，先缩小行为到微小版本
- 如果是 motivation 不足，用存在性追问连接深层价值
- 如果是 prompt 不足，帮助绑定现有习惯或创造环境提示
