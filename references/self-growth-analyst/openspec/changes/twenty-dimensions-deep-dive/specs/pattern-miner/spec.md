## ADDED Requirements

### Requirement: 跨维度关联规则挖掘器 (Pattern Miner)

通过手工+启发式方法，在每次分析中主动记录观察到的跨维度行为关联，当同类观察累积到阈值时，将经验性关联固化为可查阅的规则。

**注意**：本引擎不执行自动挖掘算法，而是依赖 Analyst Agent 在分析时的主观观察和判断，记录形式为结构化的经验性规则。

---

#### Scenario: 分析时记录关联观察

- **WHEN** Analyst Agent 在单次分析中发现两个维度之间存在明显的伴随关系（如某用户某次健身后的几天内执行力评分明显提升）
- **THEN** Analyst Agent 将该关联记录到 `pattern_observations.md`，格式：`[观察] 维度A → 维度B （观察日期: YYYY-MM-DD）`

---

#### Scenario: 3次同类观察触发规则创建

- **WHEN** 同一关联观察（相同的维度A → 维度B）在不同日期出现3次或以上
- **THEN** Analyst Agent 创建正式规则，写入 `cross_dimension_rules.md`，格式：`[规则] 前提行为 → 预期结果 （证据：YYYY-MM-DD1, YYYY-MM-DD2, YYYY-MM-DD3）`

---

#### Scenario: 规则格式规范

- **WHEN** 创建跨维度规则时
- **THEN** 每条规则必须包含以下字段：
  - **规则内容**：简明的"前提行为 → 预期结果"描述
  - **证据强度**：1-5星（1星=仅3次观察，3星=多次重复验证，5星=跨多用户验证）
  - **最后验证日期**：最近一次触发该规则的观察日期
  - **失效条件**：触发该规则不再成立的条件（如"前提行为中断超过2周"）
  - **来源观察**：引用具体观察记录

---

#### Scenario: 示例规则生成

- **WHEN** Analyst Agent 观察到用户在健身后的连续3天内执行力评分均有提升（日期：2026-04-12, 2026-04-19, 2026-04-26）
- **THEN** 创建规则：`[规则] 健身 → 执行力提升 （证据：2026-04-12, 2026-04-19, 2026-04-26, 证据强度: ★★☆）`

---

#### Scenario: 规则触发标注

- **WHEN** 用户的行为触发了某条已记录的跨维度规则
- **THEN** 分析洞察中自动标注该规则：`触发规则：[规则名称]（证据强度: ★★★）`，帮助Coach Agent了解行为间的潜在关联，并据此调整干预策略
