# OpenSpec Proposal: Twenty Dimensions Deep Dive

## Why

当前系统本质是一个"会提问的档案系统"——收集数据、存储画像、偶尔提问，但缺乏真正的诊断深度和行动驱动力。用户需要的是"冷酷的教练+精准导航仪"：不只记录数据，更能穿透表象发现问题根因；不只提问，更能给出可执行的下一步。

二十维度体系已成型，但支撑它的引擎、分析逻辑、知识储备和可视化能力仍停留在初级阶段。这次升级是系统从"档案室"到"教练系统"的关键跃迁。

---

## What Changes

将 self-growth-analyst 从维度覆盖型系统，进化为具备行为诊断引擎、跨维度关联挖掘、硬核知识图谱支撑的深度分析系统。核心变化：

1. **行为模型层** — 引入福格行为模型、冰山模型、大五人格作为底层诊断逻辑，使分析具备心理学深度
2. **分析引擎层** — 将模糊的"追问"升级为规则驱动的结构化诊断流程
3. **Skills Library 全面升级** — 20个维度从简单 probes 进化为完整的 skill.yaml，包含触发条件、诊断流程、干预手段和成功指标
4. **记忆系统进化** — 从能力基线存档进化为行为证据链追踪、认知偏误长期监测
5. **知识图谱** — 构建心理学+管理学双知识库，使系统"懂得为什么"，而不只是"知道是什么"
6. **KPI 仪表盘** — 20维度趋势可视化，让用户看见自己的进化路径

## Capabilities

### 行为模型层（新增）

- **fogg_behavior_model** — 福格行为模型 B=MAP 诊断引擎。当用户描述行为或习惯问题时，自动拆解为动机(Motivation)、能力(Ability)、提示(Prompt)三要素，定位失败根因并给出最小行动建议
- **iceberg_engine** — 冰山模型深层分析触发器。当对话中出现"失败/沮丧/搞砸/卡住"等关键词时，自动触发冰山下层追问（价值观、信念、恐惧、渴望），穿透行为层看系统层
- **big_five_OCEAN** — 大五人格底层背景分析。在对话中持续积累行为证据，构建用户的大五人格轮廓（开放性、尽责性、外向性、宜人性、神经质），为所有维度分析提供人格背景校准
- **pattern_miner** — 跨维度关联规则挖掘。每月自动扫描记忆数据，寻找维度间的隐藏关联（如"高执行力但低抗压"的矛盾模式、"社交智慧强但影响力弱"的落差点）

### 分析引擎层（新增 + 改造）

- **behavioral_model.md** — 硬编码福格行为模型作为核心诊断逻辑框架，所有习惯/行为类问题走统一诊断路径，确保 coach 不只提问而能给出结构化诊断
- **iceberg_engine.md** — 冰山追问触发规则。当检测到挫败感或重复失败模式时，自动激活冰山下层探测，不止步于表层原因
- **pattern_miner.md** — 跨维度关联分析任务定义。每月生成维度关联报告，发现用户自己都未察觉的能力组合或木桶短板
- **dimension_diagnostic_engine** *(改造)* — 原有诊断流程升级为引擎架构。接收用户行为输入 → 匹配行为模型 → 激活相关 skill.yaml → 输出结构化诊断而非开放式追问

### Skills Library 全面升级（改造）

- **20个维度 skill.yaml 升级** — 每个维度从 probes.md 升级为完整 skill，包含：trigger_on（触发条件）、diagnosis_flow（诊断流程）、intervention（干预手段）、success_metrics（成功指标）
  - 认知、学习力、自我认知、执行力、决策力、情绪智力、沟通力、影响力、商业嗅觉、战略思维、创造力、抗压与韧性、时间管理、社交智慧、领导力、理财力、健康管理、政治智慧、信息整合、目标驱动
- **skill_orchestrator** — 维度间协调器。当单一维度 skill 触发时，自动评估是否需要联动其他维度（如"执行力弱"可能触发"自我认知"和"抗压"的相关 skill）

### 记忆系统进化（改造）

- **capability_baseline.json** *(改造)* — 增加关键行为证据字段，从"分数记录"进化为"行为证据链"（具体事件+时间戳+情境+结果），让每一次分析有据可查
- **long_term/personal_bias_tracker.md** *(新增)* — 固有认知偏误长期追踪。记录用户反复出现的认知偏误（确认偏误、损失厌恶、虚假紧迫感等），在分析中主动提示和矫正
- **long_term/cross_dimension_rules.md** *(新增)* — 跨维度关联规则积累区。pattern_miner 发现的新关联写入此处，形成系统对用户的"个性化理解"
- **memory_consolidation_task** — 每周记忆整合任务。将当周碎片记忆归类、提炼、更新到长期记忆，防止记忆膨胀失效

### 知识图谱（新增）

- **psychology_knowledge_base** — 心理学知识库。涵盖行为分析学（ABC模型、Fogg模型）、认知行为疗法（认知重构技术）、动机心理学（自我决定论），在诊断时提供循证依据而非泛泛而谈
- **management_knowledge_base** — 管理学知识库。涵盖 GTD 方法论、复盘方法论（GRAI/ORID）、商业模式画布，使分析能触达工作方法和商业思维层面
- **weekly_dimension_report** — 每周维度趋势报告自动生成任务。基于本周对话数据，更新各维度评分变化、标注异常波动、提出下周追踪建议
- **knowledge_retrieval_engine** — 诊断时实时检索知识库，将相关理论/案例注入分析输出，提升建议的专业度和可信度

### 个人 KPI 仪表盘（新增）

- **dimension_trend_dashboard** — 20维度趋势图。每周/每月自动更新，以折线图展示各维度变化轨迹，高亮异常波动
- **open_questions_tracker** — 待追踪问题标注系统。用户在对话中提出的待验证假设或计划，自动汇入仪表盘待办项，下次对话主动跟进
- **dashboard_renderer** — 可视化输出组件。将 JSON 数据渲染为用户可读的图表+摘要格式，支持在飞书等平台展示

## Impact

**对系统：**
- 从"问答记录器"升级为"具备诊断引擎的教练系统"，分析不再浮于表面
- 知识图谱使系统输出有理论根基，从"我觉得"进化为"根据XXX理论/模型，你的情况是..."
- 行为证据链使每次诊断可追溯、可验证、可迭代

**对用户：**
- 获得的不只是"你在这个维度几分"，而是"你在这个行为上失败的原因是什么，下次可以怎么做"
- 每月收到个人进化报告，看见真实的成长轨迹而非模糊感觉
- 跨维度关联发现让用户理解"为什么我总在某类事情上出问题"——这是档案系统永远给不了的洞察

**对维护者：**
- skill.yaml 体系使每个维度的诊断逻辑显性化、易维护、易扩展
- 知识库模块化，新理论/模型可独立注入，不影响核心架构
- 行为证据链为后续 AI 微调或个性化模型提供高质量训练数据
