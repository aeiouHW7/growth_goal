## ADDED Requirements

### Requirement: 新增3个财富专用维度
在20维基础上新增3个维度，不删除现有维度，保持可扩展结构。

#### Dimension 1: investment_judgment（投资判断力）
- **定义**: 对投资标的（股票/基金/数字资产/实体资产）的分析能力和判断准确度
- **锚定值**: 7（百富榜级别需要强投资能力）
- **杠杆权重**: 2.5（投资能力对财富的乘数效应极大）
- **追踪场景**: 用户提到买了什么、卖了什么、投资决策的理由

#### Dimension 2: leverage_awareness（杠杆识别能力）
- **定义**: 识别并使用杠杆（资本/劳动力/代码/媒体）的能力
- **锚定值**: 8（百富榜的人都是杠杆高手）
- **杠杆权重**: 3.0（最高杠杆维度）
- **追踪场景**: 用户提到融资、招募团队、用工具放大产出、建立系统

#### Dimension 3: asset_diversity（收入来源多元化）
- **定义**: 主动收入/被动收入/投资收入的比例和稳定性
- **锚定值**: 6（不是百富榜核心，但影响财富稳定性）
- **杠杆权重**: 1.5
- **追踪场景**: 用户提到工资、项目收入、被动收入、投资收益的比例变化

### Requirement: 财富维度初始化
新增维度从 baseline 开始，current_score 初始为 0 或合理初值。

#### Scenario: 初始化
- **WHEN** 系统启动且用户首次数据收集
- **THEN** investment_judgment=0, leverage_awareness=0, asset_diversity=0（从0开始追踪）

### Requirement: 财富维度融入主流程
新增维度必须能和原有20维一起出现在：
- 周报/月报
- scoring_engine 双轨计分
- intervention_trigger 的触发条件

#### Scenario: 综合报告
- **WHEN** 生成周报
- **THEN** 同时包含原有20维 + 3个财富维度，统一格式
