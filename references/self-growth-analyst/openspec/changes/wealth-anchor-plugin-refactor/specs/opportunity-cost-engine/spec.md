## ADDED Requirements

### Requirement: 机会成本分析引擎
opportunity_cost_engine.md 分析用户"没做什么"带来的财富损失。

#### Scenario: 时间分配分析
- **WHEN** 用户本周花了15小时在"通勤+低效会议"，3小时在"公众号内容生产"
- **THEN** 输出：你在"通勤+会议"上损失了约12小时的财富杠杆时间

### Requirement: 财富杠杆时间识别
引擎必须能区分"高杠杆活动"和"低杠杆活动"。

高杠杆活动：学习核心技能、创作内容、建立人脉、投资理财、运动健身
低杠杆活动：通勤、无目的刷手机、无效社交、纯消费娱乐

#### Scenario: 周时间审计
- **WHEN** 每周日运行机会成本分析
- **THEN** 输出时间分配报告：高杠杆小时数 / 低杠杆小时数 / 空白时间

### Requirement: ROI对比建议
引擎必须给出"如果你把这3小时花在做X而不是Y，财富收益会提高Z"的建议。

#### Scenario: ROI对比
- **WHEN** 用户花了3小时刷抖音 vs 应该写公众号文章
- **THEN** 机会成本分析：刷抖音的财富期望值 ≈ 0，写公众号的财富期望值 ≈ 高（长期品牌积累）

### Requirement: 不评判原则
机会成本分析必须客观陈述，不带情绪，不让用户感到被指责。

#### Scenario: 输出语气
- **WHEN** 输出本周机会成本分析
- **THEN** 使用"你本周的时间分配如下"而不是"你浪费了X小时"
