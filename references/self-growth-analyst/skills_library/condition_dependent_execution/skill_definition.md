# 条件依赖型执行 — 诊断脚本 (Condition Dependent Execution Skill)

## 基本信息
- skill_name: 条件依赖型执行校准
- dimension: execution
- version: 1.0
- created_at: 2026-05-04
- 基于: retrieval_index pattern "条件依赖型执行"（新发现，与"知道但不做"同源）
- 同源模式: "知道但不做"(11次)

## trigger_conditions（触发条件，满足任一即触发）
- pattern: "条件依赖型执行" frequency_gte: 1
- pattern: "知道但不做" frequency_gte: 2
- keywords: ["等", "如果", "看看心情", "再说", "到时候看"]
- behavior: 把外部条件（测试通过/心情好/天气好）当作行动触发，而非自己设定触发

## diagnosis_flow（诊断流程，按顺序执行）

### Step 1: 依赖类型诊断
**问题**: "你说的'等XX'，那个XX是你能控制的吗？"
**输出**:
- 外部可控（他人决定/测试结果）→ 依赖外部，不可靠
- 外部不可控（天气/他人情绪）→ 纯看运气
- 内部可控（自己的心情/状态）→ 把主动权交给情绪

### Step 2: 历史追踪
**问题**: "上一次你说'等XX就做'，XX发生之后，你真的做了吗？"
**输出**: 做了 / 没做 / 部分做了

### Step 3: 根因定位
**方法**: iceberg engine
**问题**: "如果你可以现在就开始，不需要任何条件，你会开始吗？"
**输出**:
- 能开始 → 不是能力问题，是触发设计问题
- 不能开始 → 深层原因（恐惧/完美主义/意义模糊）

## intervention_options（干预选项，根据诊断结果选择）

### 如果是触发设计问题
**Prompt**: "你在等一个外部信号。不是因为不想做，是因为你把触发权交给了别人/外部条件。"
**行动**: "自己设定一个触发：不管测试结果如何，今天打开Obsidian，创建草稿标题。"

### 如果是深层原因（恐惧/完美主义）
**Prompt**: "你不是在等条件成熟。你是在等一个'保证成功的承诺'。"
**行动**: "降低预期：只做最烂的一版，不需要任何保证。"

### 如果是意义模糊
**Prompt**: "你还没想清楚这件事做成了对你意味着什么。"
**行动**: "写下笔记整合成功的3个最直接的好处。"

## verification_method（验证方法）
- 检查用户下次输入是否提到"我打开了Obsidian"或"我开始做了"
- verification_keywords: ["打开了", "开始了", "写下来了", "创建了草稿"]
- 48小时内无行动 → 再次触发

## escalation（升级路径）
- 如果连续3次触发，仍然条件依赖 → 升级为深度挑战："你是在'谨慎'还是'害怕面对结果'？"
- 如果用户有明显情绪（焦虑/恐惧）→ 保护模式，跳过常规干预
