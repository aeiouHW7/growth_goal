# 任务时间估算乐观 — 诊断脚本 (Task Time Optimism Diagnostic Skill)

## 基本信息
- skill_name: 任务时间估算乐观校准
- dimension: time_management
- version: 1.0
- created_at: 2026-05-04
- 基于: retrieval_index pattern "任务时间估算乐观"

## trigger_conditions（触发条件，满足任一即触发）
- pattern: "任务时间估算乐观" frequency_gte: 2
- keywords: ["30分钟", "1小时就够了", "很快", "花不了多少时间", "应该能完成", "估计"]
- behavior: 提到任务完成时间，但实际未在预估时间内完成

## diagnosis_flow（诊断流程，按顺序执行）

### Step 1: 时间感知诊断
**方法**: 检查用户对单任务时间预估 vs 实际执行时间的比例
**问题**: "你估计30分钟，实际花了多久？"
**输出**: 乐观系数 = 实际时间 / 预估时间（>1.5为严重乐观）

### Step 2: 规划模式诊断
**方法**: 行为模式分析
**问题**: "你通常在规划时怎么估算时间？"
**输出**: 
- 粗略估算（没有拆解任务步骤）
- 理想状态估算（假设所有条件都完美）
- 忽略了"摩擦时间"（找东西/切换/回复消息）

### Step 3: 后果评估
**问题**: "时间预估错了，对你造成了什么影响？"
**输出**: 
- 耽误别人/领导（任务延误）
- 打乱后续计划（时间挤压）
- 自我愧疚（"我又低估了"）

## intervention_options（干预选项，根据诊断结果选择）

### 如果粗略估算问题
**Prompt**: "你没有把任务拆成具体步骤，所以无法准确估算。"
**行动**: "下次规划时，把任务拆成3个最小步骤，估算每个步骤的时间。"

### 如果理想状态估算问题
**Prompt**: "你假设所有条件都完美，但实际总会有摩擦。"
**行动**: "预估时间 × 1.5，这是你的实际时间。"

### 如果是习惯性低估
**Prompt**: "这不是能力问题，是习惯。你每次都低估，根源是没有建立反馈机制。"
**行动**: "每晚复盘一次：今天哪件事花的时间超过了预期？差距多少？"

## verification_method（验证方法）
- 检查用户下次提到任务时间时是否有"乘1.5"的表达
- verification_keywords: ["乘", "1.5", "多估了", "保险起见"]
- 7天内无改善 → 再次触发

## escalation（升级路径）
- 如果连续3次触发，仍然低估 → 升级为深度挑战："你是在'保持乐观'还是'拒绝承认自己慢'？"
