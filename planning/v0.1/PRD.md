# 人生教练 — 产品需求文档 (PRD)

## 1. 产品定位

基于用户人生愿景的全能型 AI 人生教练。它了解你的全部生活状态，
帮你把人生愿景拆解为可执行的目标体系，在日常中持续指导、复盘、调整。

> **愿景驱动**: 不是管理你给的 todo，而是理解你想成为什么样的人，倒推需要做什么。

---

## 2. 核心概念

### 2.1 人生愿景 (Life Vision)
用户对自己理想人生的描述。例如：
> "过上能自由选择上班、自由选择休假的家庭幸福人生"

愿景是教练一切行动的**最高指引**。所有目标的设定都要回溯到愿景。

### 2.2 人生领域 (Life Domains)
基于愿景拆解出的关键领域。每个用户不同，例如：
- 💼 事业/职业
- 💰 财务
- 🏋️ 健康
- 👨‍👩‍👧 家庭
- 📚 学习/成长
- 🤝 社交/人脉
- 🎯 副业/创作

### 2.3 目标层级 (Goal Hierarchy)

```
人生愿景
    ↓ 拆解
人生领域目标 (5-10年)
    ↓ 拆解  
年度目标 (可量化)
    ↓ 拆解
月度计划
    ↓ 拆解
每日任务
```

### 2.4 教练对话 (Coaching Session)
用户与教练的每一次对话。教练在其中：
- 了解用户的近况、想法、困惑
- 给出分析和建议
- 帮助用户拆解目标、调整计划
- 记录关键决策和洞察

## 3. 用户故事

### 3.1 初次使用
> 作为新用户，我告诉教练我的人生愿景，教练帮我分析我现在处于什么阶段，
> 应该在哪些领域努力，帮我制定初步的目标框架。

### 3.2 每日复盘
> 作为用户，我每天向教练汇报今天做了什么，
> 教练帮我分析完成情况、指出认知偏误、识别行为模式、给出改进建议。

### 3.3 目标拆解
> 作为用户，我说"我想在半年内把副业收入做到 1 万/月"，
> 教练帮我把这个愿望拆解成具体的月度计划、关键里程碑、需要培养的能力。

### 3.4 进度检查
> 作为用户，我问"我的视频项目怎么样了"，
> 教练查看历史数据，告诉我已经停滞了 X 天，分析卡点和原因，给出推进建议。

### 3.5 定期总结
> 作为用户，教练每周/每月自动给我出一份总结，告诉我这个周期
> 的进步、退步、模式趋势、调整建议。

### 3.6 主动提醒
> 当教练发现有价值的信息时（如某个目标连续多日无进展、某个模式出现了 3 次以上），
> 主动在复盘时提醒用户。

## 4. 功能需求

### F1: 人生愿景管理
- 用户可表达/修改自己的人生愿景
- 教练基于愿景自动分析关键领域和差距
- 愿景作为所有目标设定的最高参考

### F2: 目标体系管理
- 支持从愿景到每日任务的多层级目标
- 教练辅助拆解（用户表达愿望 → 教练建议目标结构和度量方式）
- 目标状态追踪和自动提醒

### F3: 每日复盘与分析
- 用户通过飞书发送每日复盘
- 教练分析：完成总结、偏差分析、认知偏误、行为模式、能力评分
- 保持现有的结构化报告格式（兼容 web 展示）
- **预处理**：信号评分（规则引擎先判断输入是否够丰富，再决定是否追问）

### F4: 教练对话
- 用户可随时向教练提问、讨论、寻求建议
- 教练基于用户画像、历史数据、目标体系给出个性化建议
- 对话上下文保持（同一话题可延续讨论）

### F5: 定期总结
- 周报：本周完成率、模式趋势、调整建议
- 月报：月度复盘、目标进度、能力变化趋势
- **预处理**：聚合数据由后端 API 提供，教练负责分析和洞察

### F6: 执行诊断
- 当用户某目标持续无进展时，教练主动分析原因
- 使用 Fogg 行为模型（B=MAP）诊断：动机不足、能力不足还是提示不足
- 给出对应干预建议

### F7: 认知偏误提醒
- 在用户表达中检测偏误模式（计划谬误、自我美化、事后合理化等）
- 温和地指出并帮助用户纠正
- **预处理**：偏误检测先用规则引擎做第一遍，教练再深度分析

## 5. 交互流程

### 5.1 日常复盘流程

```
用户在飞书发消息（复盘内容）
    ↓
Bridge/Agent 判断是否需追问（信号规则引擎）
    ↓ 如需追问
追问 1-2 轮补充信息
    ↓
教练 Agent 接收完整输入
    ↓
Agent 加载上下文（curl API）:
  - 今日计划
  - 近 7 天复盘
  - 活跃行为模式
  - 认知偏误历史
  - 能力评分基线
  - 人生愿景 & 目标体系
    ↓
Agent 生成分析报告（JSON + 自然语言回复）
    ↓
写回 API:
  - 存储 structuredReport
  - 更新行为模式
  - 记录偏误日志
  - 更新能力评分
    ↓
用户收到教练回复 + 评分提示
```

### 5.2 目标拆解流程

```
用户: "我想半年内副业做到 1 万/月"
    ↓
Agent 分析:
  - 当前副业状态（读 API）
  - 和历史目标/模式的关联
  - 时间可行性
    ↓
Agent 给出拆解建议:
  - 推荐月度里程碑
  - 建议关键能力培养路径
  - 建议每周工作量
    ↓
用户确认/调整
    ↓
Agent 写入 API:
  - 创建/更新年度目标
  - 创建月度计划
  - 创建初期每日任务
```

## 6. 预处理规则（节省 token）

以下逻辑由规则引擎/后端处理，不经过大模型：

| 功能 | 处理方式 | 现有实现 |
|------|---------|---------|
| **信号评分** | 规则引擎判断输入完整度 | `SignalDepthService` ✅ |
| **追问生成** | 根据缺失维度生成定向问题 | `buildFollowUp()` ✅ |
| **精力诊断** | 3 问标准问卷 | `ENERGY_DIAG_QUESTIONS` ✅ |
| **体态训练询问** | 简单 yes/no | Bridge 现有逻辑 ✅ |
| **偏误初筛** | 正则词典匹配第一遍 | `BiasDetectionService` ✅ |
| **模式粗略匹配** | bigram Jaccard 相似度 | `PatternService` ✅ |

这些预处理输出作为 context 传给 agent，agent 在已有结果基础上做深度分析，
而不是从零开始推理。

### 7.1 现有表（16 张，保持不变）
User, LifeGoal, YearlyGoal, MonthlyPlan, DailyPlan,
DailyReview, WeeklyReview, MonthlyReview,
AIAnalysis, AIAnalysisFeedback, AIReflection, AISuccessCase,
BehaviorPattern, CognitiveBiasLog, CapabilityScore, AnalysisSession

### 7.2 需要新增的字段/表

**User 表扩展字段（可选）：**
- `lifeVision` (text) — 用户的人生愿景描述
- `lifeVisionUpdatedAt` (datetime)

**新表：CoachSession（可选）：**
- 记录每次教练对话的关键要点
- 用于跨会话记忆和上下文恢复

---

## 8. 非功能需求

- **响应速度**: 飞书对话应在 10s 内开始回复
- **token 节省**: 预处理 + 结构化 context，减少大模型 token 消耗
- **数据一致**: 所有分析结果通过 API 写入，Web 端直接读取展示
- **渐进增强**: 第一期先实现核心能力，后续迭代增强

---

## 9. 第一期范围（MVP）

1. coach agent 基本人设和能力定义 ✅
2. 日常复盘分析（替代现有 Bridge + claude -p -） 
3. 目标拆解对话（愿望 → 目标）
4. 预处理规则保留（信号评分、精力诊断、体态询问）
5. Web 端保持现有仪表盘不变

后续迭代：
- 定期总结（周/月报）
- 主动提醒
- 能力趋势可视化
- 对话历史管理和回顾
